# 格子战线 (Grid Frontline) — High Level Plan

## 游戏概述

**类型：** 策略 + 即时战斗 + 卡牌构筑 + 自走棋式战斗  
**引擎：** Godot 4.x + C#  
**核心体验：** 在格子地图上放置建筑，编排兵种组合，经历"防守 → 发展 → 反攻"的战斗弧线。

---

## 地图结构

```
[玩家区域 6×8] → [走廊/交战区] → [敌方基地 6×N]
  建筑放置(格子)     2D自由战斗       关卡变化
```

- **玩家区域（6行×8列）**：格子系统，放置经济/军事/防御建筑，最左列为玩家基地
- **走廊/交战区**：2D自由移动区域，双方士兵在此交战，长度随关卡调整
- **敌方基地（6行×N列）**：关卡预设布局，摧毁基地核心即胜利

> ⚠️ 行（Row）仅用于建筑放置。士兵进入走廊后为 **2D自由移动**，不受行限制。

---

## 核心系统

### 经济
- 唯一资源：**金币**
- 经济建筑周期产出，放置建筑消耗金币

### 建筑（三大类）

| 类别 | 功能 | 示例 |
|------|------|------|
| 经济建筑 | 产出金币 | 农场、矿场 |
| 军事建筑 | 生成士兵 | 兵营、马厩 |
| 防御建筑 | 阻挡/伤害敌军 | 城墙、箭塔 |

**防御建筑分级：**
- **低级防御**（城墙、路障）：前期抵挡敌军进攻，争取经济发展时间
- **高级防御**（箭塔、炮台）：攻击范围可覆盖走廊，后期仍有火力价值

### 士兵系统（自走棋式）

- 由军事建筑周期生成
- **2D自由移动**，自动寻敌、自动战斗
- 不同兵种有不同属性（生命、攻击、移速、攻速）
- **每个兵种拥有技能**，类似自走棋：

| 触发方式 | 说明 | 适合兵种 |
|----------|------|----------|
| 蓄能释放 | 攻击/受击积累能量，满了释放 | 大多数兵种 |
| 冷却计时 | 固定间隔自动释放 | 辅助/治疗 |
| 血量阈值 | 低于一定血量触发 | 坦克/狂战士 |
| 死亡触发 | 阵亡时触发效果 | 自爆兵/亡语类 |
| 被动光环 | 持续生效，不需触发 | 指挥官/旗手 |

> 设计原则：技能 **可预期但不可精确控制**，玩家通过兵种组合间接影响战局。

### 集结派兵机制

```
兵营产兵 → 士兵进入集结区等待 → 玩家手动派出 / 满员自动出发
```

**关键规则：集结区有士兵等待时，兵营暂停生产。**

这制造了核心策略张力：
- **攒兵 = 停产**：等更好的阵容，但浪费了生产时间
- **速派 = 散兵**：保持产能，但小队战斗力弱
- 玩家需要持续决策派兵时机，避免"挂机等赢"
- 可配合法术卡弥补小队不足（给弱队加增益）

### 卡牌构筑系统

**卡牌分两类：**

| 类型 | 说明 | 使用方式 |
|------|------|----------|
| **建筑卡** | 代表一种可放置的建筑 | 放在玩家区格子上，消耗金币 |
| **法术卡** | 主动技能（火球、治疗、加速等） | 对走廊/战场指定位置释放 |

- **战前**：从收藏中选一组卡牌带入战斗（建筑卡+法术卡混搭）
- **战中**：随时使用，金币足够即可
- 卡牌通过通关、商店等途径获取和升级

### 敌方系统
- 有自己的建筑和士兵，机制与玩家 **不对称**
- 敌方兵种有独特技能和行为模式
- 关卡不同，敌方策略不同
- 战前展示部分敌方情报，让选卡有据可依

---

## 策略深度（三层决策）

```
第1层：造什么建筑 — 经济/军事/防御的比例与位置
第2层：派什么兵种 — 技能组合、兵种克制、集结时机
第3层：法术卡时机 — 主动干预战场关键时刻
```

---

## 战斗流程

```
选卡 → 开局建设 → 防守敌军 → 发展经济 → 编排兵种反攻 → 摧毁敌方基地
                                              ↑
                                    持续决策：攒兵 vs 速派
                                    持续操作：法术卡干预
```

**胜利条件：** 摧毁敌方基地核心  
**失败条件：** 己方基地被摧毁

## 开发阶段

| 阶段 | 目标 | 关键交付 |
|------|------|----------|
| **Phase 1 — 核心原型** | 格子地图、建筑放置、产金币、产兵、2D自由移动 | 可玩的最小原型 |
| **Phase 2 — 战斗系统** | 士兵AI与战斗、技能系统、集结派兵、防御建筑、胜负判定 | 完整单局战斗 |
| **Phase 3 — 卡牌与关卡** | 建筑卡+法术卡、卡组选择、关卡配置、敌方AI | 多关卡可玩 |
| **Phase 4 — 打磨** | UI/UX、音效动画、数值平衡、更多兵种/卡牌、存档 | 可发布版本 |

## 技术架构

```
Godot 4.x + C#

场景树:
Main
├── UI Layer（HUD、卡牌选择、金币、集结区）
├── GameBoard
│   ├── PlayerZone (6×8 Grid)
│   ├── Corridor (2D自由移动区)
│   └── EnemyZone (6×N)
├── UnitManager（士兵管理、寻路、AI）
├── BuildingManager（建筑管理、生产队列）
├── RallyManager（集结区管理、派兵控制）
├── SkillSystem（技能触发、效果处理）
├── SpellManager（法术卡释放、效果处理）
├── EconomyManager（金币系统）
└── LevelManager（关卡数据、敌方AI）

核心类:
  GridCell, Building, Unit, UnitSkill, Card (BuildingCard / SpellCard),
  Deck, RallyZone, LevelData, GameManager, EnemyAI

数据驱动:
  建筑/士兵/技能/卡牌属性 → Godot Resource
  关卡配置 → JSON / Resource
```

## 实现难点提醒

1. **士兵2D寻路与AI**：自由移动需要良好的寻路和目标选择逻辑
2. **视觉可读性**：走廊大量单位混战需要清晰的视觉层次
3. **数值平衡**：防守→反攻的节奏转折点需要精心调参
